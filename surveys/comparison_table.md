## Column Guide

- **Variable**: 논문에서 조절하는 변수. 예: window size, input length, model, period, batch, exit.
- **Trigger**: 적응 또는 선택을 유발하는 기준. 예: anomaly score, machine condition, criticality, workload, slack, offline profile.
- **Runtime/Offline**: runtime adaptation인지, offline profiling/selection인지 구분한다.
- **RT Constraint**: deadline, latency, jitter, schedulability, PREEMPT_RT 등 실시간 제약 고려 여부.
- **Relation to My Work**: 본 연구 주장 또는 원고에서 어떻게 쓸 수 있는지.
- **Gap**: 본 연구와 비교했을 때 해당 논문이 다루지 않는 부분.
