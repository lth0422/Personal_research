## 0. 핵심 결론 요약

1. **스케줄링 이론을 명시적으로 사용한 진동 결함 진단 논문은 2021~2026 범위에서 사실상 0편** — 후보 1편도 원문 판정 필요. 압도적 다수는 모델 경량화로 평균 latency만 줄이고 real-time이라 칭하는 best-effort 연구.
2. 실시간 스케줄링 이론이 활발히 적용되는 곳은 **GPU 기반 edge server 추론 서빙**이며, 임베디드 MCU와 RTOS 계층에는 거의 전무 — 최신 서베이가 이를 "OS and Scheduling Bottleneck"으로 지목.
3. Elastic scheduling 이론 계보에서 **Orr 2020의 discrete utilization 모델**이 본 연구의 이론적 거점 — period와 computation 양축 탄력성 + 이산 mode 집합이 우리 (W,H,M) 구조와 정확히 대응하며, 교수님 질문 "어떻게 guarantee 하나"의 답 구조를 제공.
4. 스코프 축소 결정: **Pi Zero 2W + PREEMPT_RT + SCHED_FIFO, CPU 전용, 이산 mode 집합, 사전 admission 기반 보장** — 각 결정에 서베이 근거 존재.

## 4. 산출물 2 — Elastic Scheduling 기법·응용·가정 정리

교수님 지시: "단순 참고문헌이 아니라 각 연구의 주된 가정과 접근을 싹 정리."

| 연구 | 접근 | **주된 가정** | 응용 | 우리 적용성 |
| --- | --- | --- | --- | --- |
| Buttazzo & Abeni 2000 | feedback 기반 rate 적응 — 실측 실행시간 모니터링, **a priori WCET 불필요** | 주기 task, 선형 탄성계수, EDF, 실행시간 측정 가능 | RT kernel 구현 | WCET 추정이 어려운 DNN에 유리한 관점 |
| Marinoni & Buttazzo 2007 | DVS + elastic 결합, **discrete mode** 3전략 | discrete speed level, WCET 기지, 최고속 feasible | S.Ha.R.K. | 이산 mode 운영의 선례 |
| Burgio 2010 | TDMA bus + elastic 협동 | MPSoC 공유 버스, 중앙 master, EDF | ERIKA RTOS | 멀티코어 확장 시 참고 |
| Tian 2011 | QoC 기반 elastic | 제어 품질 지표 연동 | 제어 시스템 | 진단 품질 U 연동의 유추 선례 |
| Wang·He·Wonham 2016 | **TDES supervisory control** — 안전한 reconfiguration 경로 전체를 사전 계산 | T_min/T_max 존재, 이산 사건 모델, 형식 검증 | 산업 자동화 | 전환 안전성의 형식적 접근 — mode change protocol 대안 |
| Orr 2020 ★ | **discrete candidate utilization** — 연속 범위 대신 task별 이산 mode 집합. **period 탄력 + computational 탄력 결합** | federated, 병렬 task, NP-hard → pseudo-poly 알고리즘 | RTHS controller 전환 | **이론적 거점.** mode별 (workload, period) = 우리 (C(W,M), H) |
| Salman 2021 | 2-level compositional — local elastic 우선, 실패 시 bandwidth 요청 | uniprocessor, implicit deadline, PRM | 로봇 컨트롤러 | 계층 구조 참고 |
| Baruah 2023 | **PDA 직접 사용** — utilization test의 constrained-deadline 한계 극복 | D<T, EDF, sporadic, 정수 period | CPS 가변 수요 | D<T 필요해질 때의 도구 |
| Sudvarg 2024 | harmonic 제약 + **offline lookup → online O(1)** | harmonic period, total order, implicit deadline | FIMS, ORB-SLAM3 | online 결정 비용 최소화의 모범 |

**종합 관찰**
- 이산 mode + 사전 계산 + 저비용 online 결정이 최신 elastic의 공통 패턴 — 우리 mode bank 설계와 정합
- 모든 계보에서 **탄력 변수는 period 또는 workload 수준** — 입력 크기 W가 실행시간·정확도를 동시에 바꾸는 구조, 그리고 **기계 상태 트리거**는 부재
- FLEX 등 elastic-DNN은 batch·fusion 탄력, GPU 전제 — (W,H,M) 동시 탄력 사례 없음

## 5. 갭 종합 — 본 연구의 자리

| 연구 계보 | 가변 변수 | 트리거 | 플랫폼 | 도메인 |

| Classic elastic (Buttazzo) | period T | 시스템 부하 | 이론 | 일반 |
| Sudvarg 계보 (RTAS·RTSS 24) | period, subtask | 부하 | 멀티코어 | 일반 |

| AIL / ADW | window | 없음 — offline | — | fault diagnosis |
| **본 연구** | **W + H + M 동시** | **기계 상태 q + slack S** | **SBC + PREEMPT_RT, CPU 전용** | **진동 FD** |

## 7. 문제 정식화 (서베이 반영 갱신)

```
mode 집합:  M = {m₁, …, m_k},  m = (W, H, M_model),  각 (W, M_model)은 사전 학습 인스턴스
하한:       W_phys ≤ W          결함 정보 보존 — AIL 물리식 / ADW 분리도
상한:       W ≤ W_sys(D)        deadline — C ∝ W² 실측에서 도출

feasible mode set (오프라인 사전 검증):
  F = { m ∈ M :  Σ_other C_i/T_i + C(W,M_model)·fs/H ≤ U_lub }
  → Orr 2020 판정 구조. 모든 전환은 F 안에서만 발생 = schedulability 보장의 1차 답

online 정책 (O(1) lookup — Sudvarg 패턴):
  π : (기계 상태 q, slack S) → m* ∈ F
  q 낮음 → 경량 mode로 slack 확보 / q 높음 → F 내 최정밀 mode
```

### KCC 데이터 기반 utilization 계산 (교수님 지시 이행)

| W | C_max 실측 | T = W/fs | U = C/T | 판정 |
| --- | --- | --- | --- | --- |
| 512 | 40.3ms | 64ms | **0.63** | feasible |
| 1024 | 129.8ms | 128ms | **1.01** | 초과 — KCC 실측 deadline miss와 정합 |
| 2048 | 460.3ms | 256ms | **1.80** | 심각 초과 |

- **관찰**: C ∝ W², T ∝ W → **U ∝ W.** 정밀 mode로 갈수록 U가 선형 증가 — "정밀해질수록 schedulability가 위협받는" 상충 구조가 수치로 확인됨
- 함의: W만 키워서는 feasible 불가 → **H를 함께 늘려야** (W,H) 결합 조절이 성립. 3차원 탄력의 필요성이 자체 데이터로 입증됨
- average U는 실측 분포(σ=0.009ms 등)로 산출 예정 — max 곡선과 병기

## 8. 오픈 질문 및 논의 요청
